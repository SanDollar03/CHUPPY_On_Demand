# ondemand専用プロジェクト 引き継ぎメモ

このプロジェクトは **ondemand専用の最小構成** です。
手動画面 / 自動画面 / ナレッジ画面は削除済みで、**ondemand.html 関連だけを残しています**。

```text
ondemand_only_project/
├─ app.py

├─ ondemand_queue.py
├─ prompt.md
├─ templates/
│  └─ ondemand.html
└─ static/

   └─ style.css
```

## プロジェクトの目的
ユーザーが自分の担当フォルダにファイルを追加すると、
公平に分散したキューで順番待ちさせながら、
**Markdown変換 → Difyナレッジ登録** を自動実行する ondemand 専用アプリです。

## 重要な作業ルール
- 機能変更は慎重に行う
- 既存動作を壊さず、必要な修正だけを入れる
- **変更時は完全ファイルで返す**
- **Goと言うまで記憶してください**
- **Goと言うまでコメントアウトは不要**

## 現在の主要仕様
1. 画面は `ondemand.html` のみ
2. `app.py` 起動時から常時フォルダ監視
3. 差分ファイルを検出したらキューへ投入
4. キューはフォルダ公平性を保つラウンドロビン方式
5. 順番が来たファイルは Markdown 変換 → Dify ナレッジ登録
6. 変換スタイルは RAG向け Markdown
7. chunk separator は `***`
8. Dify 登録設定は `separator / max_tokens / search_method = hybrid_search`
9. 失敗時は最大5回リトライ
10. 最終的にエラーになったものは **元ファイル / Markdown / Difyドキュメントを自動削除**
11. 内容一覧からの削除でも **元ファイル / Markdown / Difyドキュメント / 同一キュー** を削除
12. Dify に通信できない時は内容一覧の削除ボタンを無効化
13. Dify 側に同一 Markdown ファイル名が既にある場合は差分なしとしてスキップ
14. 操作ログ `ondemand_queue.log` は軽量形式
   - `timestamp`
   - `folder_name`
   - `file_name`
   - `status`

## ナレッジ名ルール
Lv5 フォルダの相対パスから自動判定します。
- 先頭に `Chu_`
- パス中の `元データ` は除外
- `/` は `_` に置換

例:
`P/PJ/PJ4/元データ/生産`
→ `Chu_P_PJ_PJ4_生産`

## Markdown保存先ルール
例:
`\\172.27.23.54\disk1\Chuppy\P\PJ\PJ4\元データ\生産\1.pdf`
→
`\\172.27.23.54\disk1\Chuppy\P\PJ\PJ4\マークダウン形式\生産\1.md`

## 現在のUI仕様・実装済みポイント
- 固定ヘッダー + 左固定サイドバー
- ondemand画面のみ
- 左ツリーはかなりコンパクト寄りのレイアウト
- フォルダツリーは **Lv1展開済みが初期状態**
- ツリーは **Lv4の `元データ` を見せず**、見た目上は Lv3 から Lv5 を選ぶ形
- ツリーの展開記号は **展開前 `>` / 展開後 `∨`**
- フォルダ名テキストをクリックしても展開/閉じる操作が可能
- ツリー右側に **配下対応ファイル数** を `人材育成(30)` のように表示
- ファイル数の文字色は赤
- Lv5 フォルダ名の前に絵文字を付与
  - `安全` → `💞 安全`
  - `品質` → `🏅 品質`
  - `生産` → `🏭 生産`
  - `保全` → `🪛 保全`
  - `環境` → `🌳 環境`
  - `原価` → `💰 原価`
  - `人材育成` → `🏫 人材育成`
- 内容一覧とアップロード待ちキューは **タブ切替表示**
- 内容一覧には **状態列** を表示
  - `登録済`
  - `未登録`
  - `エラー`
  - `登録中`
- 状態は常に最新状況へ追従するよう定期更新
- 内容一覧の **名前 / 状態 / 更新日時 / サイズ** は昇順・降順ソート可能
- 規定値は **更新日時の降順**
- 内容一覧とキューのファイル名左に拡張子アイコン表示
- ドラッグ&ドロップ欄はコンパクト表示
- ドラッグ&ドロップ欄に **追加可能拡張子** と **1ファイル最大100MB** の注意書きを表示

## アップロード制約
- **非対応拡張子は内容一覧で非表示**
- **非対応拡張子はアップロード不可**
- **非対応拡張子はキュー追加不可**
- **100MBを超えるファイルはアップロード不可**（フロント側・サーバー側の両方で拒否）

## 現在の主な担当ファイルの役割
### app.py
- Flask ルート
- アプリ起動
- キュー/監視インスタンス生成

### ondemand_core.py
- 共通設定
- ファイル抽出
- パス/フォルダ探索
- 状態判定
- Markdown変換
- Dify登録/削除
- キャッシュ
- ログ出力

### ondemand_queue.py
- フェアキュー制御
- リトライ制御
- フォルダ監視ワーカー
- 自動削除制御

### templates/ondemand.html
- ondemand画面の骨組み
- ヘッダー / サイドバー / ドロップ領域 / タブ / 内容一覧 / キュー

### static/ondemand.js
- ツリー描画
- API通信
- アップロード
- 内容一覧の状態表示
- ソート
- タブ切替
- キュー表示更新
- 削除操作

### static/style.css
- 固定レイアウト
- コンパクト表示
- ツリー見た目
- テーブル見た目
- 状態ラベル見た目
